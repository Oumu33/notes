# <font style="color:rgb(0, 0, 0);">作用</font>
<font style="color:rgb(51, 51, 51);">当在一个镜像指定了 HEALTHCHECK 指令后，用其启动</font>容器<font style="color:rgb(51, 51, 51);">，初始状态会为 starting，在 HEALTHCHECK 指令检查成功后变为 healthy，如果连续一定次数失败，则会变为 unhealthy</font>

# <font style="color:rgb(0, 0, 0);">格式</font>
```bash
# 设置检查容器健康状况的命令
HEALTHCHECK [选项] CMD <命令>

# 如果基础镜像有健康检查指令，使用这行可以屏蔽掉其健康检查指令
HEALTHCHECK NONE
```

# <font style="color:rgb(0, 0, 0);">注意</font>
<font style="color:rgb(51, 51, 51);">和 CMD, ENTRYPOINT 一样，HEALTHCHECK 只可以出现一次，如果写了多个，只有最后一个生效</font>

# <font style="color:rgb(0, 0, 0);">HEALTHCHECK 返回值</font>
<font style="color:rgb(51, 51, 51);">决定了该次健康检查的成功与否：0：成功；1：失败；2：保留（不要使用这个值）</font>

# <font style="color:rgb(0, 0, 0);">为啥要用 HEALTHCHECK</font>
+ <font style="color:rgb(51, 51, 51);">在没有 HEALTHCHECK 指令前，Docker 引擎只可以通过容器内主进程是否退出来判断容器是否状态异常</font>
+ <font style="color:rgb(51, 51, 51);">很多情况下这没问题，但是如果程序进入死锁状态，或者死循环状态，应用进程并不退出，但是该容器已经无法提供服务了</font>
+ <font style="color:rgb(51, 51, 51);">在 1.12 以前，Docker 不会检测到容器的这种状态，从而不会重新调度，导致可能会有部分容器已经无法提供服务了却还在接受用户请求</font>
+ <font style="color:rgb(51, 51, 51);">自 1.12 之后，Docker 提供了 HEALTHCHECK 指令，通过该指令指定一行命令，用这行命令来判断容器主进程的服务状态是否还正常，从而比较真实的反应容器实际状态</font>

# <font style="color:rgb(0, 0, 0);">HEALTHCHECK 支持的选项</font>
+ <font style="color:rgb(51, 51, 51);"> ：两次健康检查的间隔，默认为 30 秒；</font>

<font style="color:rgb(51, 51, 51);">--interval=<间隔></font>

+ <font style="color:rgb(51, 51, 51);"> ：健康检查命令运行超时时间，如果超过这个时间，本次健康检查就被视为失败，默认 30 秒；</font>

<font style="color:rgb(51, 51, 51);">--timeout=<时长></font>

+ <font style="color:rgb(51, 51, 51);"> ：当连续失败指定次数后，则将容器状态视为 unhealthy ，默认 3 次</font>

<font style="color:rgb(51, 51, 51);">--retries=<次数></font>

# <font style="color:rgb(0, 0, 0);">示例</font>
<font style="color:rgb(51, 51, 51);">假设有个镜像是个最简单的 Web 服务，希望增加健康检查来判断其 Web 服务是否在正常工作，可以用 curl 来帮助判断其 </font>

## <font style="color:rgb(0, 0, 0);">dockerfile</font>
```javascript
FROM nginx
RUN apt-get update && apt-get install -y curl && rm -rf /var/lib/apt/lists/*
HEALTHCHECK --interval=5s --timeout=3s \
  CMD curl -fs http://localhost/ || exit 1
```

## <font style="color:rgb(0, 0, 0);">执行 docker build</font>
```javascript
docker build -f HEALTHCHEK.dockerifle -t myweb .
```

## <font style="color:rgb(255, 255, 255);background-color:rgb(105, 113, 145);"></font><font style="color:rgb(0, 0, 0);">执行 docker run</font>
```javascript
docker run -d --name web -p 80:80 myweb
```

+ <font style="color:rgb(51, 51, 51);">如果健康检查连续失败超过了重试次数，状态就会变为 unhealthy</font>
