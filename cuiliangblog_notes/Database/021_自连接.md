# 自连接

---

1. 自连接是连接的一种用法,但并不是连接的一种类型,因为他的本质是把一张表当成两张表来使用。
2. mysql有时在信息查询时需要进行对自身连接（自连接），所以我们需要为表定义别名。
3. 我们举例说明，下面是商品采购表，我们需要找到采购价格比小张高的所有信息。
+ ![](https://via.placeholder.com/800x600?text=Image+a5fec2b75ee06b03)
+ 一般情况我们看到这张表我们第一时间用语句进行操作：
+ SELECT * FROM shoping WHERE price>2
+ 分步查询
+ SELECT  price FROM shopping WHERE name='小张' //得出price查询结果为2
+ SELECT  * FROM shopping WHERE price>2
+ 我们可以获取到如下表信息：
+ ![](https://via.placeholder.com/800x600?text=Image+8d0aa30f4c0d89c7)
4. 与采用自连接的方式相比，这种方法需要对中间结果进行人工干预，显然不利于程序中的自动处理操作。
+ 自连接方式：
+ SELECT  b.*
+ from shopping as a,shopping as b
+ where  a.name='小张'
+ and  a.price<b.price
+ 获取到如下表信息：　　
+ ![](https://via.placeholder.com/800x600?text=Image+a3f832cc08b25f16)
5. 注意点：
+ 别名  a,b虽然名称不同，但是同一个表，定义别名的目的是更方便在自身进行删选。
