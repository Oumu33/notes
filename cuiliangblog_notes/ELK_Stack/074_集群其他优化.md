# 其它参数优化
+ <font style="color:rgb(62, 62, 62);">Linux参数调优修改系统资源限制#单用户可以打开的最大文件数量，可以设置为官方推荐的65536或更大些echo</font>
+ <font style="color:rgb(62, 62, 62);">设置内存熔断参数，防止写入或查询压力过高导致OOM</font>

<font style="color:rgb(51, 51, 51);">indices.breaker.total.limitindices.breaker.request.limitindices.breaker.fielddata.limit</font>

+ <font style="color:rgb(62, 62, 62);">索引、分片等信息都被维护在clusterstate对象中，由master管理，并分发给各个节点。当集群中的index/shard过多，创建索引等基础操作会变成越来越慢，而且master节点的不稳定会影响整体集群的可用性。</font><font style="color:rgb(51, 51, 51);"></font>

<font style="color:rgb(62, 62, 62);">可以考虑：</font>

+ <font style="color:rgb(62, 62, 62);">拆分独立独有的master节点</font>
+ <font style="color:rgb(62, 62, 62);">降低数据量较小的index的shard数量</font>
+ <font style="color:rgb(62, 62, 62);">把一些有关联的index合并成一个index</font>
+ <font style="color:rgb(62, 62, 62);">数据按某个维度做拆分，写入多个集群</font>
