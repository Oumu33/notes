# 修改节点pod个数上限

> 来源: Kubernetes
> 创建时间: 2023-08-15T11:53:56+08:00
> 更新时间: 2026-01-11T09:11:16.768144+08:00
> 阅读量: 1439 | 点赞: 0

---

默认情况下，<font style="color:rgb(18, 18, 18);">单台节点能调度的Pod数为110个，如果机器配置较高，可考虑调整此参数</font>

# <font style="color:rgb(18, 18, 18);">修改配置文件</font>
```bash
# vim /etc/sysconfig/kubelet
KUBELET_EXTRA_ARGS="--max-pods=500"
```

# 重启kubelet服务 
```bash
systemctl restart kubelet 
```

# 检查
```bash
kubectl describe node work1 |  grep -i "Capacity\|Allocatable" -A 6
Capacity:
  cpu:                192
  ephemeral-storage:  9298787068Ki
  hugepages-1Gi:      0
  hugepages-2Mi:      0
  memory:             1056273036Ki
  pods:               200
Allocatable:
  cpu:                192
  ephemeral-storage:  8569762147680
  hugepages-1Gi:      0
  hugepages-2Mi:      0
  memory:             1056170636Ki
  pods:               500 # 检查pod上限是否更改成了500个
```
